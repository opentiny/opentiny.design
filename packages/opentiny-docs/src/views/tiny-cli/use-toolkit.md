<h1 id='useToolkit'>使用套件</h1>

---
<h2 id='introduce'>TinyCLI 套件介绍</h2>

TinyCLI 套件是特定业务的整体解决方案模板，提供全链路开发解决方案，从最初本地项目初始化，到开发过程中的调试、构建和最终的代码发布上线等功能都由套件提供。

使用 TinyCLI 套件，可以享受到一致化的开发体验。tiny-cli 对前端相关的开发生命周期进行了抽象，将所有开发过程划分为六个阶段，每个阶段由一个固定的子命令加以支持。不管开发哪种业务体验完全一致。不再需要记忆复杂冗长的命令，记住以下 tiny-cli 的几个固定子命令即可。

<h2 id='kitCommand'>套件固定命令</h2>

<h3 id='init'>tiny init: 初始化项目结构</h3>

使用套件进行业务开发时，一般 tiny init 是您需要执行的第一个命令，执行该命令后默认为会显示一个可用套件列表菜单待您选择，根据您的业务模型选择合适的套件后即会根据套件的设置初始化您的目录结构和一些基本配置信息。

<h3 id='start'>tiny start: 开启本地开发环境</h3>

使用 tiny start 命令可以在本地项目目录快速启动一个开发调试服务（如果是开发 Web 页面，一般套件会实现一个 Web Server）。同时会监听项目文件的变化，进行实时编译。

<h3 id='test'>tiny test: 执行测试任务</h3>

如果您的项目 /test 文件夹中提供了相关的单元测试脚本，执行 `tiny test` 时将自动运行这些单元测试。

<h3 id='build'>tiny build: 源码构建与打包</h3>

如果您想获取本地源代码构建后的内容，执行 `tiny build` 即可。tiny 会将源码进行压缩、合并最终输出到 `build`目录

<h3 id='publish'>tiny publish: 发布项目代码</h3>

执行 `tiny publish` 后将执行项目仓库中对应开发分支下的代码发布任务。根据不同套件所支持业务的特点，最终将构建后的代码发布到对应的环境。

<h3 id='tinyHelp'>tiny help: 查看套件帮助</h3>

在项目根目录下，执行`tiny help`命令，会显示当前项目的套件帮助信息。不同套件可能有一些不同的命令。若不在项目根目录，执行该命令，只会显示 tiny的帮助信息。


<h2 id='case'>例子</h2>

1. 初始化项目

	```bash
	# 创建并进入项目文件夹
	$ mkdir my-project && cd $_
	
	# 初始化tiny的开发环境
	$ tiny init tiny
	```
	
2. 开启本地环境

	```bash
	# 开启tiny的开发环境
	$ tiny start
	```
	
	
3. 项目编译及打包

	```bash
	# 打包tiny项目的到指定的目录
	$ tiny build
	```

4. 发布项目代码

	```bash
	# 将编译后的代码发布至CDN环境
	$ tiny publish
	```
